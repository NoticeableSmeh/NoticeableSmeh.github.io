Prizm: What I did

What it is

Prizm is a mobile puzzle game where the player directs light beams through mirrors and splitters to produce the correct colour at one or more goals. The project demonstrates a complete gameplay system with deterministic reflections, splitting, colour mixing, and level progression.

How I built it

  • Implemented beam paths using a precomputed list of reflection points so the LineRenderer and the gameplay logic always share the exact same path.
  • Built mirror and splitter mechanics. Mirrors reflect beams deterministically, and splitters spawn a second beam that follows its own path while the original continues.
  • Implemented colour mixing so multiple beams can combine to produce target colours for goal checks.
  • Created goal validation and level logic that verify win conditions when all targets match their required colours.
  • Developed Unity editor tools and level templates to speed up puzzle iteration and testing.
  • Debugged edge cases such as missing beams, incorrect reflections, and split beams not being removed correctly.

Why I chose this approach

I aimed for deterministic, easy-to-test logic and a clear visual representation that always matches the game rules. Separating path calculation from rendering makes the beam behavior predictable and easier to debug. The editor tools were created to keep iteration fast, which is important in puzzle development.

What I take to the next project

  • Create reusable tools early since they save a lot of time during level creation.
  • Keep a single source of truth for gameplay and visualisation to avoid mismatches.
  • Add better logging and telemetry to reproduce and fix bugs faster in complex interactions.
  • Plan simple automated tests for core mechanics such as reflections, splitters, and colour mixing.

What I would do with more time

  • Add more advanced level-authoring tools, such as previews, additional templates, and quick profiling for balance testing.
  • Improve visual feedback for colour mixing and collisions so players understand the rules faster.
  • Optimize the beam system for handling many simultaneous beams and for performance on lower-end mobile devices.
  • Create a lightweight playtesting pipeline with feedback forms and simple telemetry to improve levels based on player data.

Short summary

I developed the core mechanics in Prizm: deterministic reflections, splitters, colour mixing, goal validation, and editor tools for fast level iteration. The work shows both technical skill and the ability to complete a full gameplay system.
